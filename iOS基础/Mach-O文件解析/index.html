Mach-O 文件解析

名称 描述
Header mach_header_64 结构体，包括支持的 CPU 架构、文件类型、Load Commands 的大小和数量等
Load Commands 加载命令表，每个 Load Command 固定包括命令类型和大小（struct load_command），依据命令类型又有不同的结构体类型
__TEXT,__text 只有可执行的机器码
__TEXT,__stubs 符号桩。本质上是一段会直接跳入 __DATA,__la_symbol_ptr 的表对应项指针指向的地址的代码。
__TEXT,__stub_helper 辅助函数。上述提到的 __DATA,__la_symbol_ptr 的表中对应项的指针在没有找到真正的符号地址的时候，都指向这。
__TEXT,__gcc_except_tab
__TEXT,__objc_methname OC 方法名 C String，还包括实例变量名 C String
__TEXT,__cstring 去重后的 C 字符串
__TEXT,__objc_classname OC 类名 C String
__TEXT,__objc_methtype OC 方法类型 C String，还包括实例变量类型 C String
__TEXT,__const 初始化过的常量，待研究
__TEXT,__ustring 非英文字符串转 Unicode 编码，Unicode 有 1-4 个字节，如何确定字符是 2 个字节的中文还是 4 个字节的中文？
__TEXT,__entitlements 权限 XML
__TEXT,__unwind_info 用于存储处理异常情况信息
__TEXT,__eh_frame 调试辅助信息
__DATA,__nl_symbol_ptr(x86) Non Lazy Binding 的指针表，加载的同时动态链接符号
__DATA,__got(ARM) Non Lazy Binding 的指针表，加载的同时动态链接符号；数据符号指针，比如枚举、常量；以及需要在加载时链接的函数符号，比如 dyld_stub_binder
__DATA,__la_symbol_ptr Lazy Binding 的指针表，第一次调用时动态链接符号；函数符号指针，一开始指向 __TEXT,__stub_helper
__DATA,__mod_init_func C++ 初始化函数，在 main 之前调用，直接指向 __TEXT,__text 中的代码实现
__DATA,__const 没有初始化过的常量，待研究
__DATA,__cfstring[5] CFString 字符串结构体对象，其中 String 内容指向 __TEXT,__cstring
__DATA,__objc_classlist OC 类列表，指向 __DATA,__objc_data 中的 class_t 结构体对象
__DATA,__objc_nlclslist 实现了 +load 的 OC 类列表，指向 __DATA,__objc_data 中的 class_t 结构体对象
__DATA,__objc_catlist OC Category 列表，指向 __DATA,__objc_const 中的 category_t 结构体对象
__DATA,__objc_nlcatlist 实现了 +load 的 OC Category 列表，指向 __DATA,__objc_const 中的 category_t 结构体对象
__DATA,__objc_protolist OC Protocol 列表，指向 __DATA,__data 中的 protocol_t 结构体对象
__DATA,__objc_imageinfo
__DATA,__objc_const OC 常量表，method_list_t、ivar_list_t、class_ro_t、prop_list_t、category_t 等等结构体对象
__DATA,__objc_selrefs OC Selector 引用到的指针列表，指向 __TEXT,__objc_methname 的 C 字符串
__DATA,__objc_protorefs OC Protocol 引用到的指针列表，指向 __DATA,__data 中的 protocol_t 结构体对象
__DATA,__objc_classrefs OC Class 引用到的指针列表，指向 __DATA,__objc_data 中的 class_t 结构体对象；另含 Foundation 等系统库中的 OC Class 引用指针（未正确指向）
__DATA,__objc_superrefs OC Super 引用到的指针列表，指向 __DATA,__objc_data 中的 class_t 结构体对象
__DATA,__objc_ivar ivar_t 中的 offset，4 字节存储，当前 ivar 变量基于对象地址的偏移量
__DATA,__objc_data class_t 结构体对象表
__DATA,__data protocol_t 结构体对象表
Dynamic Loader Info Rebase Info、Binding Info、Weak Binding Info、Lazy Binding Info、Export Info
Function Starts
Data in Code Entries
Symbol Table 所有的符号表，存储符号对应的 C String 在 String Table 的索引、符号类型（N_SECT 本地符号、N_UNDF 未定义符号等等）、section Index、符号指针指向地址
Dynamic Symbol Table 动态连接的符号表，存储符号在 Symbol Table 中的索引，按顺序依次对应__TEXT,__stub、__DATA,__got、__DATA,__la_symbol_ptr
String Table C String 字符串表
Code Signature 签名

！！！以上内容未列全！！！而且未包含 Swift 二进制相关 segment、section

另外 otool 命令能加 v 就加 v，可以尽可能打印出更详细的（符号化）内容

-v print verbosely (symbolically) when possible

nm 命令能加 m 就加 m，可以打印出本地符号的 segment 和 section，其他符号也有更人性化的表示

-m Display the N_SECT type symbols (Mach-O symbols) as (segment_name, section_name) followed

​ by either external or non-external and then the symbol name. Undefined, common, absolute

​ and indirect symbols get displayed as (undefined), (common), (absolute), and (indirect),

​ respectively. Other symbol details are displayed in a human-friendly manner, such as “[no

​ dead strip]”. nm will display the referenced symbol for indirect symbols and will display

​ the name of the library expected to provide an undefined symbol. See nlist(3) and <mach-

​ o/nlist.h> for more information on the nlist structure.
